# 公开仓库前检查清单

这份清单面向准备把 FFMeet 发布到 GitHub 或其他代码托管平台的维护者。

## 1. 先确认授权方式

- 仓库根目录已有 `LICENSE`
- 当前许可证是否符合预期的公开范围
- 如需商用授权或 OEM 合作，是否已有额外说明

## 2. 先确认敏感文件不会进入仓库

- `.env` 不在仓库中
- `database/database.sqlite` 不在仓库中
- `storage/logs/*` 不在仓库中
- `storage/debugbar/*` 不在仓库中
- `storage/app/*` 与媒体临时文件不在对外交付包中
- `public/storage/*` 不在对外交付包中

## 3. 先确认仓库首页可直接读

- `README.md` 已替换为项目级说明
- 安装步骤可直接执行
- 文档入口清晰可见
- 当前版本定位与限制写清楚

## 4. 先确认开发者配套资料

- `CHANGELOG.md` 已存在
- `.github/ISSUE_TEMPLATE/*` 已存在
- `.github/pull_request_template.md` 已存在
- `.github/SECURITY.md` 已存在
- `docs/` 目录可直接阅读

## 5. 先确认运行和安装口径

- 默认开发数据库写清楚
- 生产推荐数据库写清楚
- 默认安装不附带固定管理员账号
- 创建首个管理员的命令写清楚
- 依赖安装使用锁定版本而非自动升级

## 6. 先确认发布包可复现

- `composer package:source` 可运行
- 生成 `.tar.gz` 包
- 生成 `.sha256` 校验文件
- 验包时不包含 `.env`、本地数据库、日志、调试文件和上传文件

## 7. 先确认产品边界说清楚

- 真实支付渠道是否已正式接入
- 未完成项是否在 README 或文档中明确
- 对外是否应标注为观察版、预览版或正式版

## 8. 最后再公开仓库

在以上项目全部确认之前，不建议直接把当前工作目录原样公开。
