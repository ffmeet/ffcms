# 1.0 发布清单

## 定位

`1.0` 版本可以作为“生产观察版”发布，用真实流量验证：

- 前台内容浏览与主题展示
- 会员中心闭环
- 作者、评论、活动、商品、订阅与订单链路
- 后台设置、首页位配置、缓存中心与主题切换

当前最明显的未完成项，是“真实支付链路”仍未做正式网关联调。

## 发布前必须完成

1. 关闭生产环境中的模拟支付入口。
2. 明确 webhook 校验方案，未完成前不得把诊断型回调当作正式支付回写。
3. 检查生产环境 `.env`：
   - `APP_ENV=production`
   - `APP_DEBUG=false`
   - `APP_URL` 为正式域名
   - `MAIL_*` 为真实 SMTP
   - `QUEUE_CONNECTION`、缓存与日志策略符合生产要求
4. 执行一次内容、会员、活动、订单、评论、后台配置的完整回归。
5. 准备人工回滚方案：
   - 数据库备份
   - 资源构建产物备份
   - `.env` 备份

## 建议上线顺序

1. 先部署代码与静态资源。
2. 运行缓存预热与视图缓存。
3. 后台确认当前主题、首页位置位、品牌设置和路由设置。
4. 用测试账号验证前台与会员中心。
5. 再开放真实用户测试。

## 发布后观察重点

1. 首页主题位是否按后台配置正常取数。
2. 登录、注册、找回密码邮件是否真实送达。
3. 会员中心是否存在跳后台、空链接、错误跳转。
4. 上传、媒体库、活动报名与订单状态是否一致。
5. 日志中是否出现大量 404、500、CSRF、邮件或 webhook 异常。
