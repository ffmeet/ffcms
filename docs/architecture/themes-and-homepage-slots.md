# 主题与首页位置系统

## 主题机制

当前站点支持主题切换，核心由 [SiteTheme.php](/Volumes/小芳侠/网站代码库/帝国 cms/ffmeet/laravel-ecms/app/Support/SiteTheme.php:1) 统一管理：

- 当前主题读取
- 视图解析
- 首页位置位蓝图
- 主题预览

## xiaofang 首页位置位

`xiaofang` 首页已经从“文案命名模块”改成“位置编号系统”。

当前编号：

- `01` 第一屏左栏：头条 + 下方简讯
- `02` 第一屏中栏：两条精选内容
- `03` 第一屏右栏：最新文章列表
- `04` 第二屏左栏：文化卡片主区
- `05` 第二屏右栏：订阅入口 + 活动列表
- `06` 第三屏：创作灵感区
- `07` 第四屏：继续阅读区

## 为什么采用编号

原因很直接：

- 运营文案会变
- 模块标题会改
- 但页面结构位置相对稳定

所以系统以“位置”而不是“标题文案”来定义首页配置。

## 配置规则

每个位置位可配置：

- 分类来源（多选）
- 取数上限
- 排序方式

特殊说明：

- `05` 号位是系统位，不走文章分类配置。
- `04` 号位如果只配置一个分类，就只显示一个区块，不会再自动补第二个区块。

## 兼容与 fallback

当前前台首页仍保留 fallback，用于兼容空配置和旧数据。  
这能降低空白页风险，但会增加维护复杂度。

长期建议：

- 等生产配置稳定后，把 `xiaofang` 首页逐步改成“强依赖位置位配置”的硬结构。
